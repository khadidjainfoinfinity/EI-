Sujet blanc n°1 (10 questions)

XML · DTD · XPath · XQuery · DOM — durée conseillée : 1h30.

Q1 — XQuery

Soit : element livre {attribute id {"L1"}, attribute id {"L2"}, "XML"}

1) Pourquoi cette requête génère-t-elle une erreur ? 2) Corrigez-la.

1) Un élément ne peut pas avoir deux attributs du même nom (id deux fois).
2) Renommer l'un ou fusionner : element livre {attribute id {"L1"}, attribute ref {"L2"}, "XML"} → <livre id="L1" ref="L2">XML</livre>
Q2 — XQuery (ordre attribut / contenu)

Soit : element personne {"Bonjour", attribute lang {"fr"}}

1) Pourquoi cette requête génère-t-elle une erreur ? 2) Corrigez-la.

1) Dans un constructeur d'élément, les attributs doivent venir AVANT tout autre contenu (texte ou sous-éléments). Ici le texte « Bonjour » est placé avant l'attribut → erreur (« un nœud attribut ne peut pas suivre un nœud non-attribut »).
2) Mettre l'attribut en premier : element personne {attribute lang {"fr"}, "Bonjour"} → <personne lang="fr">Bonjour</personne>
Q3 — XPath : . vs text()

XML : <p>Le <b>chat</b> dort bien</p>

Expliquer ce que retourne chaque expression :
//p[contains(., 'bien')]  et  //p[contains(text(), 'bien')]

contains(., 'bien') → . = tout le texte de p (« Le chat dort bien », y compris après </b>) → « bien » présent → retourne <p>.
contains(text(), 'bien') → text() = le premier nœud texte direct (« Le »). « bien » est après <b>, dans un autre nœud texte → ne retourne rien.
Q4 — XPath : positions

Expliquer la différence entre //item[1] et (//item)[1].

//item[1] = « le item en 1ʳᵉ position chez chaque parent » → peut renvoyer plusieurs nœuds.
(//item)[1] = on collecte tous les item puis on prend le 1ᵉʳ de la liste globale → toujours 1 seul.
Q5 — XQuery : écrire une requête FLWOR

Fichier bibliotheque.xml : <livre><titre>…</titre><prix>…</prix><annee>…</annee></livre>…

Écrire une requête XQuery qui retourne les titres des livres dont le prix > 50, triés par année décroissante, le tout dans un élément racine <resultat>.

element resultat {
  for $l in doc("bibliotheque.xml")//livre
  where $l/prix > 50
  order by $l/annee descending
  return element titre { $l/titre/text() }
}
Q6 — DTD : contenu mixte

XML : <note>Bonjour <gras>Paul</gras> !</note>
DTD : <!ELEMENT note (#PCDATA, gras)>

Cette ligne DTD génère une erreur. Corrigez-la.

note a un contenu mixte (texte « Bonjour … ! » + sous-élément gras). Forme obligatoire :
<!ELEMENT note (#PCDATA | gras)*>
Q7 — DTD : attribut ID

XML : <produit ref="P 100"/>  avec  <!ATTLIST produit ref ID #REQUIRED>

1) Pourquoi erreur ? 2) Donnez deux corrections différentes.

1) Un attribut de type ID ne peut pas contenir d'espace (la valeur « P 100 » en a un).
2) (a) changer le type : <!ATTLIST produit ref CDATA #REQUIRED> · (b) enlever l'espace : ref="P100"
Q8 — DOM : que va afficher ce code ?
<cours>
  <module code="M1">Algo</module>
  <module code="M2">Reseaux</module>
</cours>
NodeList m = doc.getDocumentElement().getElementsByTagName("module");
for(int i=0; i<m.getLength(); i++){
  System.out.println(m.item(i).getFirstChild().getNodeValue());
  System.out.println(((Element)m.item(i)).getAttribute("code"));
}

Que va afficher ce code ? Justifier.

Pour chaque module : le texte (fils #text) puis l'attribut code (après cast en Element) :
Algo M1 Reseaux M2
getElementsByTagName ne ramène que les éléments module (pas de #text parasite), donc getFirstChild().getNodeValue() donne directement le texte.
Q9 — DOM : erreur

Sur le même <cours>, on écrit :

NodeList n = doc.getDocumentElement().getChildNodes();
for(int o=0; o<n.getLength(); o++)
   System.out.println(n.item(o).getFirstChild().getNodeValue());

1) Pourquoi ce code génère-t-il une erreur ? 2) Comment la corriger ?

1) getChildNodes() ramène aussi les nœuds #text parasites (sauts de ligne/espaces de l'indentation). Sur un nœud #text, getFirstChild() vaut null → getNodeValue() sur null → NullPointerException.
2) Filtrer ces parasites avant d'accéder au fils :
if(n.item(o).getNodeName() != "#text") System.out.println(n.item(o).getFirstChild().getNodeValue());
Q10 — XQuery : eq vs =

Soit : doc("arbres.xml")//arbre/hauteur eq 30

1) Pourquoi cette requête génère-t-elle une erreur ? 2) Comment l'écrire pour obtenir un résultat booléen ?

1) eq exige une seule valeur typée. Or //arbre/hauteur est une séquence (plusieurs hauteurs) et non typée (xs:untypedAtomic) → erreur.
2) Utiliser la comparaison générale = (gère les séquences + conversion) : //arbre/hauteur = 30 (vrai s'il existe au moins une hauteur = 30). Ou caster un seul nœud : xs:int(//arbre[1]/hauteur) eq 30.

Q2 — XPath : axes (écrire l'expression)

XML composé de <mot pos="…">…</mot> (étiquetage grammatical).

Écrire l'expression XPath qui sélectionne tous les mot de POS NOM qui sont précédés immédiatement d'un mot de POS DET.

//mot[@pos='DET']/following-sibling::mot[1][@pos='NOM']
ou, en partant du NOM : //mot[@pos='NOM'][preceding-sibling::mot[1]/@pos='DET']. Le [1] garantit le voisin immédiat.
Q3 — XQuery : is et <<

XML fic.xml : <r><x>1</x><y>2</y><x>1</x></r>

Donner et justifier le résultat de :
//x[1] is //x[2]  et  //x[1] << //x[2]

//x[1] is //x[2] → false : is compare l'identité ; ce sont deux nœuds différents (même si même valeur 1).
//x[1] << //x[2] → true : << compare l'ordre ; le 1ᵉʳ x apparaît avant le 2ᵉ.
Q4 — XQuery : expliquer une construction

personne a pour enfants <nom>Dupont</nom> puis <prenom>Ali</prenom>.

element { doc("p.xml")//personne[1]/name(*[1]) }
        { doc("p.xml")//personne[1]/*[1]/text() }

Expliquer ce que produit cette requête.

Nom de l'élément = name(*[1]) = nom du 1ᵉʳ enfant de personne[1] = « nom ».
Contenu = *[1]/text() = texte du 1ᵉʳ enfant = « Dupont ».
Résultat : <nom>Dupont</nom> (on reconstruit un élément nommé d'après le 1ᵉʳ enfant, contenant son texte).
Q5 — encoding obligatoire ?

Un document commence par <?xml version="1.0"?> (sans encoding), est encodé en UTF-8 et contient le mot « café ».

L'attribut encoding est-il obligatoire ici ? Justifier.

Non, il n'est pas obligatoire. L'encodage par défaut en XML est UTF-8/UTF-16 ; comme le document est justement en UTF-8, le parseur lit correctement « café » sans déclaration. (Il serait nécessaire seulement pour un encodage non par défaut, ex. iso-8859-1.)

Q7 — DOM : sortie d'un parcours récursif

XML (sur une seule ligne) : <a>X<b>Y<c>Z</c></b></a>

public static void L(Node n){
  for(int o=0; o<n.getChildNodes().getLength(); o++){
     Node z = n.getChildNodes().item(o);
     if(z.getNodeName()=="#text") System.out.println(z.getNodeValue());
     L(z);
  }
}   // appelé avec getDocumentElement()

Que va afficher ce code ? Justifier.

Le code descend récursivement a → b → c et imprime chaque nœud #text rencontré :
X Y Z
Pas de blancs parasites car tout est sur une seule ligne (aucun espace entre balises).
Q8 — DTD : écrire la DTD
<commande num="42">
  <client>Ali</client>
  <article>Livre</article>
  <article>Stylo</article>
</commande>

Écrire la DTD complète de ce document.

<!ELEMENT commande (client, article+)>

<!ELEMENT client (#PCDATA)>

article+ car il y en a au moins un (ici deux).

Sujet blanc n°3 (6 questions express)

Révision rapide tous thèmes.

Q1 — XQuery : ordre attribut / élément

Soit : element racine {attribute n {"1"}, element a {}, attribute m {"2"}}

1) Pourquoi erreur ? 2) Corrigez.

1) L'attribut m est placé après un sous-élément (element a {}). Or tous les attributs doivent précéder tout contenu → erreur.
2) Regrouper les attributs en tête : element racine {attribute n {"1"}, attribute m {"2"}, element a {}} → <racine n="1" m="2"><a/></racine>
Q2 — XPath : expliquer

Expliquer ce que localise l'expression : //BBB/*/..

//BBB/* sélectionne les enfants-éléments de chaque BBB, puis /.. remonte à leur parent → on obtient les BBB qui ont au moins un enfant-élément. (Les BBB vides ou ne contenant que du texte ne sont pas retournés.)
Q3 — XPath : sélectionner tous les attributs

Écrire l'expression XPath qui localise tous les nœuds d'attribut du document, en utilisant l'axe descendant::.

/descendant::*/@* (équivalents : /descendant::*/attribute::* · //@*)
Q4 — XQuery : sans répétition

Écrire la requête qui compte le nombre de genres distincts dans arbres.xml.

count(distinct-values(doc("arbres.xml")//genre)) (distinct-values() enlève les doublons, count() compte.)
Q5 — XQuery : if-then-else

Écrire une requête qui, pour chaque arbre de arbres.xml, affiche son nom_commun suivi de « ancien » si son annee < 1900, sinon « récent ».

for $a in doc("arbres.xml")//arbre
return if ($a/annee < 1900)
       then concat($a/nom_commun, " : ancien")
       else concat($a/nom_commun, " : récent")
Q6 — DTD : pourquoi erreur

XML : <adresse lang="ar">Alger</adresse>
DTD : <!ATTLIST adresse lang (fr|en) #REQUIRED>

Pourquoi le document n'est-il pas valide ? Comment corriger ?

L'attribut lang est défini par une énumération (fr|en) : sa valeur doit être fr ou en. Or ici lang="ar" n'est pas dans la liste → document non valide.
Correction : soit mettre une valeur autorisée (lang="fr"), soit ajouter ar à l'énumération : <!ATTLIST adresse lang (fr|en|ar) #REQUIRED>.
